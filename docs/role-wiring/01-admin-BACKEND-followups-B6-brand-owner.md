# Admin — طلب باك: تفعيل حساب مالك البراند + إرسال باسورد (B-A6)

## المشكلة
عايزين لما الأدمن يضيف **علامة تجارية (brand)** ويكتب إيميل المالك (`ownerEmail`) →
الباك يعمل للمالك **حساب دخول** ويبعتله **باسورد على إيميله**، عشان يقدر يعمل login.

## اللي حاصل دلوقتي (متأكدين منه باختبار لايف على ivory-snail)
`POST /api/v1/admin/brands` مع `ownerEmail`:
- بيرجّع **201** وبينجح، وبيخزّن `ownerEmail` **كنص بس**.
- **مبيعملش حساب يوزر** — دوّرنا في `GET /admin/users?search=<الإيميل>` رجّع `total: 0`.
- **مبيبعتش باسورد** — الرد مفهوش `ownerUserId` ولا `emailSent` ولا `tempPassword`.

يعني المالك عمره ما هيقدر يدخل. محتاجين السلوك ده يتضاف.

---

## المطلوب بالظبط

### 1) عند إنشاء براند بـ `ownerEmail`
`POST /api/v1/admin/brands`

**Request body (نفس الحالي + الإيميل):**
```jsonc
{
  "name": "اسم البراند",
  "companyId": "019eadc6-...",   // required (زي ما هو)
  "owner": "اسم المالك",          // اختياري (اسم العرض)
  "ownerEmail": "owner@example.com",
  "plan": "silver"
}
```

**المطلوب من الباك لما `ownerEmail` موجود:**
1. لو مفيش يوزر بالإيميل ده → **اعمل يوزر جديد** بدور **brand owner**، مربوط بالبراند (والشركة `companyId`).
2. ولّد **باسورد مؤقت** وابعته على إيميله (نفس ميكانيزم إيميل أدمن الشركة الموجود عندكم).
3. لو الإيميل متسجّل قبل كده → **اربط اليوزر الموجود** بالبراند (متعملش حساب جديد)، وابعتله إشعار (اختياري).
4. لو إرسال الإيميل فشل → البراند يتعمل عادي، بس `emailSent: false`.

**Response (نزوّد حقول):**
```jsonc
{
  "id": "019f...",
  "companyId": "019eadc6-...",
  "name": "اسم البراند",
  "owner": "اسم المالك",
  "ownerEmail": "owner@example.com",
  "plan": "silver",
  "subStatus": "active",
  "status": "active",

  // ↓ الجديد المطلوب:
  "ownerUserId": "019f...",   // id اليوزر اللي اتعمل/اترابط (null لو مفيش ownerEmail)
  "emailSent": true            // اتبعت الباسورد ولا لأ
}
```

**Validation:**
- `ownerEmail` لو اتبعت لازم يكون إيميل صحيح (`email`).
- لو الإيميل مربوط بدور تاني في نفس الشركة بشكل بيتعارض → رجّعوا **422** برسالة عربية واضحة.

### 2) (اختياري لكن مفيد) إعادة إرسال / reset باسورد المالك
مرآة للموجود عند الشركة، عشان لو المالك ضيّع الباسورد:
```
POST /api/v1/admin/brands/{brandId}/owner/reset-password
body: { "notify": true }
→ { "ok": true, "emailedTo": "owner@example.com", "resetAt": "..." }
```

---

## أكواد الحالة المتوقعة
| الحالة | الكود |
|---|---|
| اتعمل براند + يوزر + اتبعت إيميل | 201 (`ownerUserId` set, `emailSent:true`) |
| اتعمل براند بس الإيميل فشل | 201 (`ownerUserId` set, `emailSent:false`) |
| `companyId` ناقص | 422 |
| `ownerEmail` صيغته غلط | 422 |
| تعارض في الدور/الإيميل | 422 برسالة عربية |

## للتأكيد
- اسم الدور بالظبط للـ brand owner عندكم إيه (roleKey)؟ عشان نظبط الفرونت عليه.
- ميكانيزم إيميل الباسورد هو نفسه بتاع أدمن الشركة (`adminEmail`)؟

**حالة الفرونت:** فورم إضافة البراند جاهز، وأول ما تفعّلوا ده هنضيف حقل الإيميل ونبعت `ownerEmail` ونعرض للأدمن إن الباسورد اتبعت.
